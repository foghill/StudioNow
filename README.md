# Studio Now

An iOS app by [WORTHLESSSTUDIOS](https://worthlessstudios.org) that matches NYC artists with affordable studio spaces.

---

## iOS App

Built with SwiftUI, targeting iOS 17+.

### Screens

| Screen | Description |
|--------|-------------|
| **Onboarding** | 3-step flow: welcome → artist profile (name, discipline) → portfolio link (optional) |
| **Studio Needs Form** | Square footage range, neighborhood multi-select, monthly budget, lease start date, duration, co-tenant preference |
| **Match Results** | List/map toggle with search bar, live/cached data badge, and filter editing. Map shows price annotations; selecting a pin reveals a detail card. |
| **Space Detail** | Hero photo, stats (sqft, rent, lease term), available date, amenities, co-tenant compatibility bar, save button, and "Request This Space" CTA |
| **Dashboard** | Application status tracker, upcoming mediation sessions, support resource links |
| **Mediation & Support** | Schedule mediation sessions, resource hub (studio setup, tenant rights, co-tenant guidelines, community forums, emergency assistance) |
| **Profile** | Edit name, discipline, portfolio URL; view saved studio needs |

### Data & State

- **AppState** (`State/AppState.swift`) — central `@MainActor ObservableObject` shared across all views
- On launch, fetches from the production API at `https://studionow-production.up.railway.app/listings`; falls back to `MockData.swift` when the server is unreachable
- Needs and profile persist to UserDefaults; saved listings tracked in-memory
- **MockData** (`Data/MockData.swift`) — 32 real-ish listings (Rockella, Brooklyn/Queens/Manhattan), 24 neighborhoods, 20 art disciplines — auto-generated by the scraper pipeline, do not edit by hand

### Tech

- SwiftUI + Swift 6.0
- MapKit for neighborhood browsing
- iOS 17+ deployment target
- XcodeGen (`project.yml`) for project file generation
- Bundle ID: `com.worthlessstudios.studionow`

## Scraper

The scraper collects studio listings from multiple sources using the [Firecrawl](https://firecrawl.dev) API for LLM-based structured extraction, then normalizes and deduplicates them into a single `listings.json`.

### Sources

| Source | Priority | Notes |
|--------|----------|-------|
| rockella | high | |
| chashama | high | |
| spacefinder | high | |
| loopnet | medium | |
| nyfa | medium | |
| listings_project | medium | |
| craigslist | medium | Restricted — opt in with `--include-restricted` |
| streeteasy | low | Restricted — opt in with `--include-restricted` |

Default runs: high-priority sources only.

### Data pipeline

```
Firecrawl extraction
      ↓
Raw JSON  →  data/raw/{source}_{timestamp}.json
      ↓
normalize_listings()
  · normalize price (string → float USD/mo)
  · normalize sqft (string → int)
  · infer borough from neighborhood/address
  · generate IDs
  · deduplicate by MD5(address + size)
  · validate (must have address or title)
      ↓
data/normalized/listings.json
      ↓
generate_mockdata.py  →  StudioNow/Data/MockData.swift
```

### Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Create `.env` in the project root:

```
FIRECRAWL_API_KEY=fc-your-key-here
```

### Usage

```bash
# Recommended — bypasses system Python PATH issues
./scrape.sh run

# Run a specific source
./scrape.sh run --source rockella
./scrape.sh run --source chashama

# Run by priority level
./scrape.sh run --priority high
./scrape.sh run --priority medium

# Run all sources
./scrape.sh run --all

# Include restricted sources (Craigslist, StreetEasy)
./scrape.sh run --all --include-restricted

# Re-normalize existing raw data without re-scraping
./scrape.sh normalize

# List sources + status
./scrape.sh sources

# List raw data files
./scrape.sh credits
```

After scraping, regenerate `MockData.swift`:

```bash
.venv/bin/python scraper/generate_mockdata.py
```

Credit budget: 500 per run (warning at 80%). Rate limit delay: 2s between requests. Max retries: 3 with exponential backoff.

## API Server

A FastAPI server that serves cached listings from disk and auto-scrapes on a configurable schedule.

### Caching strategy

- On startup, loads `listings.json` into memory and serves all queries from that cache
- A background scheduler (APScheduler) runs high-priority scrapers every `SCRAPE_INTERVAL_HOURS` (default: 24)
- If the cache is fresh on startup, the scheduler waits until the interval elapses before scraping again
- If the cache is missing or stale on startup, the first scrape is triggered after 1 minute
- The iOS app fetches once per session and falls back to `MockData` when unreachable

### Start the server

```bash
./serve.sh
```

**Production:** `https://studionow-production.up.railway.app` (Railway)

**Local development:** `http://127.0.0.1:8000` — Swagger UI at `/docs`.

```bash
# Custom host/port
HOST=0.0.0.0 PORT=9000 ./serve.sh

# Scrape every 12 hours
SCRAPE_INTERVAL_HOURS=12 ./serve.sh

# Disable automatic scraping
SCRAPE_INTERVAL_HOURS=0 ./serve.sh
```

### Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/health` | Cache status, listing count, cache age, next scheduled scrape |
| `GET` | `/listings` | Query listings with filters (see below) |
| `GET` | `/listings/{id}` | Single listing by ID |
| `GET` | `/sources` | All available scraper sources |
| `POST` | `/scrape` | Trigger a background scrape (202 Accepted) |
| `GET` | `/scrape/status` | Poll scraping progress |
| `POST` | `/cache/reload` | Reload in-memory cache from listings.json without re-scraping |

### `GET /listings` query parameters

| Param | Type | Description |
|-------|------|-------------|
| `q` | string | Full-text search across title, address, neighborhood, description |
| `neighborhood` | string | Partial match, case-insensitive |
| `borough` | string | Exact: `brooklyn`, `manhattan`, `queens`, `bronx`, `staten_island` |
| `min_price` | float | Minimum monthly rent (USD) |
| `max_price` | float | Maximum monthly rent (USD) |
| `min_sqft` | int | Minimum square footage |
| `max_sqft` | int | Maximum square footage |
| `source` | string | Filter by scraper source |
| `shared_ok` | bool | Co-tenant availability |
| `limit` | int | Results per page, max 200 (default 50) |
| `offset` | int | Pagination offset (default 0) |

**Examples:**

```bash
# All Brooklyn studios under $1,200/mo
curl "http://localhost:8000/listings?borough=brooklyn&max_price=1200"

# Search by address
curl "http://localhost:8000/listings?q=1639+Centre+St"

# Small studios, co-tenant welcome
curl "http://localhost:8000/listings?max_sqft=300&shared_ok=true"
```

### `GET /health` response

```json
{
  "status": "ok",
  "listings_cached": 33,
  "cached_at": "2026-03-25T03:00:00+00:00",
  "cache_age_hours": 2.5,
  "scrape_running": false,
  "scrape_interval_hours": 24,
  "next_scheduled_scrape": "2026-03-26T03:00:00+00:00"
}
```

`next_scheduled_scrape` is `null` when `SCRAPE_INTERVAL_HOURS=0`.

### Trigger a manual scrape via API

```bash
# High-priority sources (default)
curl -X POST http://localhost:8000/scrape

# Single source
curl -X POST http://localhost:8000/scrape \
  -H "Content-Type: application/json" \
  -d '{"source": "rockella"}'

# Medium-priority sources
curl -X POST http://localhost:8000/scrape \
  -H "Content-Type: application/json" \
  -d '{"priority": "medium"}'

# Poll status
curl http://localhost:8000/scrape/status

# Reload cache after a manual scraper run
curl -X POST http://localhost:8000/cache/reload
```
